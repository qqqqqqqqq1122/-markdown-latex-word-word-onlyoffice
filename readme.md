```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <script type="text/javascript" src="https://onlyoffice.github.io/sdkjs-plugins/v1/plugins.js"></script>
    <script>
        // 插件初始化（当你点击按钮时触发）
        window.Asc.plugin.init = function() {
            
            // callCommand 用来将代码注入到文档底层运行
            this.callCommand(function() {
                var doc = Api.GetDocument();
                var range = doc.GetRangeBySelect();
                if (!range) return;
                
                var text = range.GetText();
                if (!text || text.trim() === "") return;
                
                // 核心净水器
                var cleanText = text.trim()
                    .replace(/^(\$\$?|\\\[|\\\()\s*/, "")
                    .replace(/\s*(\$\$?|\\\]|\\\))$/, "");
                
                // 删除原文本并插入公式
                range.Delete();
                doc.AddMathEquation(cleanText, "latex");
                
            }, false, true, function() {
                // 代码运行完后，立刻自我关闭，不留痕迹
                window.Asc.plugin.executeCommand("close", "");
            });
        };
    </script>
</head>
<body style="margin: 0;"></body>
</html>

```

看到报错里那个 `via 127.0.0.1`，原因非常明确：**你的 Git 撞到了本地的网络代理（比如平时用的科学上网工具）的墙上。** 它要么是找不到你代理软件的正确端口，要么是你现在没开代理，但 Git 的脑子里还记着以前的“死胡同”。

别慌，这问题极其经典，我们分两步来“通网”：

### 情况一：你现在没开代理软件（或者想直连）

如果是这样，说明你的 Git 被以前设置的“幽灵代理”给卡住了。你需要把 Git 脑子里的代理缓存强制清空。
在那个黑框框（CMD）里，依次输入这两行命令并回车：

```cmd
git config --global --unset http.proxy
git config --global --unset https.proxy

```

敲完之后，什么都不会提示（这是正常的）。你再试一下刚才那句 `git push -u origin main`。

### 情况二：你现在开着代理软件（强烈建议用这个办法）

GitHub 在国内直连极其不稳定，既然你有本地代理（`127.0.0.1`），我们不如直接把它的“油门”踩到底，告诉 Git 你的正确代理端口！

1. 先看一下你用的代理软件（比如 Clash、V2Ray 等），它的**本地端口**是多少？（Clash 默认通常是 `7890`，V2Ray 默认通常是 `10809`）。
2. 假设你的端口是 `7890`，在黑框框里依次输入并回车：

```cmd
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

```

*(注意：如果你的软件端口是 10809 或者别的，把上面代码里的 7890 换掉就行)*

设置完正确的代理口子后，再次输入发射口令：

```cmd
git push -u origin main

```

**你按这两种情况排查一下。如果是第二种，配好端口后 `push`，速度绝对是瞬间起飞！看到黑框框里弹出进度条了吗？**
